# 2026春数据库系统实验要求

## 1．实验要求

### （1）
设计并实现一个轻量型的数据库应用系统，可以根据需要选择任何的应用程序方案，有合理且有趣的应用场景（如校园图书管理系统、超市商品管理系统等等），也可以在课外实验内容基础上扩展。

### （2）
使用实体关系模型（ER图或IDEF1X图）为数据库创建概念模型，并对模型进行详细说明，包括实体的属性及数据类型、实体间的关系类型（1:1 / 1:n / m:n）、关系的约束条件（主键 / 外键 / 唯一 / 非空）和关联角色等。设计的实体和关系需要贴合应用系统的业务功能，具备合理性和完整性，系统至少包括 8 个实体。

### （3）
将创建的ER图转换为关系模型，并讨论所产生的关系数据库模式是否足够好。

### （4）
使用你认为适合的数据库管理系统（DBMS），如 MySQL、PostgreSQL 等，搭配自己熟悉的高级编程语言实现数据库应用系统的业务逻辑与交互功能。

### （5）系统的实现与功能要求

#### a.
根据业务查询需求，使用索引、设计视图，并基于实际场景验证索引对系统查询效率的提升作用。

#### b.
实现至少 1 个复杂查询功能，查询需基于 3 张及以上表的连接查询，或带分组、排序的多层嵌套查询，贴合应用系统的实际业务需求（如多条件的统计查询、关联查询）。

#### c.
在应用程序层体现完整性约束（数据库层设置主键、外键、非空、唯一、检查约束；应用程序层做参数校验），当用户执行违规操作（如插入重复主键、外键引用无效、非空字段为空等）时，能在交互界面给出明确、具体的中文警告和提示信息。

#### d.
生成足够数量的测试数据（可借助 AI 工具）测试各种功能的查询效率，分析索引和视图的作用。

### （6）
选择适配开发语言的交互方式实现系统功能演示，可选择命令行交互或图形化简单界面：

1. 命令行交互需有清晰的功能菜单进行操作。
2. 图形化界面需具备基础的操作模块（如数据增删改查、查询功能、数据统计），无需复杂的视觉设计。

## 2．时间节点

### 第1次实验课
熟悉所选 DBMS（MySQL / PostgreSQL）的基础操作，包括创建数据库、新建表、添加约束、简单 SQL 查询等；确定数据库应用系统的业务场景、功能性需求和非功能性需求。

### 第2次实验课
完成初步的逻辑数据库库表和属性的设计，完成初步物理数据库的实现。完成复杂查询功能以及视图功能的 SQL 语句设计与实现。

### 第3次实验课
根据数据建模思想检查初步实现的数据库表的合理性；完成高级语言 + 嵌入式 SQL 的代码实现，实现数据库应用系统的基础数据增删改查功能。

### 第4次实验课
验收实体关系模型（ER图或 IDEF1X 图），验收完整的系统功能，重点是实验要求（5）的所有要求。

## 3．实验报告内容

1. 实验要求，简要描述实验任务要求。
2. 需求描述，描述系统的功能性需求和非功能性需求（如性能需求等）。
3. 概念数据库设计，ER 图或 IDEF1X 图设计。详细描述实体和联系。
4. 逻辑数据库和物理数据库设计。能够解释如何从概念数据库设计逻辑数据库，并根据关系模式分析数据库。解释设计的数据库表、属性、类型和约束等。
5. 数据库创建：数据库管理系统中建立的库表。
6. 程序的实现，有必要的功能流程图，关键数据结构和函数的实现。
7. 总结和心得体会。
